Đến lập trình viên công ty Nhật Nissan còn đi copy-paste code ở trên mạng, quên không xóa dấu vết Đừng bao giờ copy những dòng code mà bạn không hiểu.Copy và paste code từ internet là công việc mà bất kỳ lập trình viên nào cũng đã từng làm. Đôi khi chúng ta có thể đặt câu hỏi trên các diễn đàn như StackOverflow.com để nhận được những đoạn code hỗ trợ từ cộng đồng lập trình viên. Okay, bạn có thể copy nhưng mà đừng copy từng chữ nếu không muốn hành vi của bạn bị lật tẩy. Và cũng đừng copy khi bạn làm trong một tập đoàn lớn như Nissan để rồi phải xấu hổ như anh chàng lập trình viên dưới đây. Anh chàng đảm nhiệm công việc phát triển ứng dụng di động NissanConnect EV vừa bị một phen bẽ mặt khi người dùng bắt quả tang anh ta copy code trên Stack Overflow vào mã nguồn của bản cập nhật ứng dụng. Dòng chú thích ghi nguồn gốc của đoạn code là từ Stack Overfow hiển thị rõ ràng trên ứng dụng. Ngoài sự cẩu thả của anh chàng lập trình viên, việc một đoạn code copy trên mạng có thể vượt qua các quy trình kiểm tra chất lượng và được triển khai như một bản cập nhật cho ứng dụng quản lý xe hơi dấy lên lo ngại trong lòng người dùng. Đến hãng xe hơi Nissan của Nhật còn làm việc tắc trách như thế thì không biết các hãng khác sẽ như thế nào? Ngoài ra, ai còn dám tin vào xe hơi kết nối và các ứng dụng đi kèm khi mà các dòng code của chúng được copy trên mạng xuống. Bài học rút ra từ sự cố này cho các lập trình viên là đừng bao giờ copy-paste những đoạn code trên mạng mà bạn không hiểu mục đích/tác dụng của nó, thứ nữa, nếu copy, hãy xóa hết dấu vết nhé...
