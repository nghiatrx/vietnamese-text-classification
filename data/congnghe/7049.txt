Microsoft vừa chuyển sang sử dụng Git và GVFS để phát triển Windows Gã khổng lồ phần mềm Microsoft vừa tuyên bố rằng phần lớn kỹ sư của họ đã chuyển sang sử dụng Hệ thống Quản lý Phiên bản Phân tán Git cho việc phát triển hệ điều hành Windows.Kho lưu trữ của Windows trên Git sẽ bao gồm khoảng 3,5 triệu tập tin với dung lượng xấp xỉ 300 GB. Tuy nhiên, do Git không được xây dựng cho những dự án quy mô lớn như thế này nên Microsoft đã phải phát triển một hệ thống mang tên Git Virtual File System (GVFS) để tận dụng những lợi thế khi sử dụng Git mà không cần phải chờ hàng giờ để một lệnh Git đơn giản được thực thi. Code cho GVFS hiện đã được tung ra theo giấy phép của MIT trên GitHub và được để mở nhằm nhận những đóng góp từ cộng đồng. Quá trình chuyển sang Git mất khoảng ba tháng. Trước đó, Microsoft đã sử dụng Source Depot để quản lý code Windows dù các nhóm code cơ sở nhỏ hơn vẫn sử dụng Team Foundation Server. Trong suốt ba tháng qua, Microsoft đã chuyển một số nhà phát triển Windows sang kho Git mới nhằm kiểm tra hệ thống. Sau đó, vào tháng 3, hãng này đã triển khai Git cho tất cả 2.000 kỹ sư thuộc nhóm Windows OneCore. Hiện tại, khoảng 3.500 trong số 4.000 kỹ sư của nhóm Windows đang sử dụng Git. Ngoài ra, Microsoft cũng tuyên bố rằng họ đã xây dựng một giải pháp Git proxy cho GVFS nhằm quản lý các vấn đề băng thông chắc chắn sẽ nảy sinh so tính chất phân bố rộng của nhóm. Một số nhà cung cấp Git đã hỗ trợ Microsoft như Atlassian, trong khi đó các hãng như Tower và GitKracken cũng đang nghiên cứu để hỗ trợ Microsoft. Theo Tech Crunch
